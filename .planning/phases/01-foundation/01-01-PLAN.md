    Create new SvelteKit project in `pwa/` directory (NOT in the repo root — the Docusaurus site lives at root and must coexist).

    **Step 1: Initialize project.** Run `npm create svelte@latest pwa` choosing the "Skeleton project" template with JavaScript (no TypeScript). Then from `pwa/`: `npm install` and `npm install -D @sveltejs/adapter-static mode-watcher @lucide/svelte svelte-check vitest @testing-library/svelte jsdom`.

    **Step 2: Create `pwa/svelte.config.js`:**
    ```javascript
    import adapter from '@sveltejs/adapter-static';
    import { vitePreprocess } from '@sveltejs/vite-plugin-svelte';

    /** @type {import('@sveltejs/kit').Config} */
    const config = {
      preprocess: vitePreprocess(),
      kit: {
        adapter: adapter({
          pages: 'build',
          assets: 'build',
          fallback: '200.html',
          precompress: true,
          strict: true
        }),
        serviceWorker: {
          register: false
        },
        paths: {
          base: ''
        }
      }
    };

    export default config;
    ```
    CRITICAL: `serviceWorker.register: false` prevents conflict with @vite-pwa (added in Plan 03). `fallback: '200.html'` enables SPA routing on Cloudflare Pages. Per RESEARCH.md Pitfall 2 and Pitfall 7.

    **Step 3: Create `pwa/vite.config.js`:**
    ```javascript
    import { sveltekit } from '@sveltejs/kit/vite';
    import { defineConfig } from 'vite';

    export default defineConfig({
      plugins: [sveltekit()],
      test: {
        include: ['src/**/*.test.{js,ts}', 'tests/**/*.test.{js,ts}'],
        environment: 'jsdom',
        globals: true
      }
    });
    ```
    NOTE: The SvelteKitPWA plugin is NOT added here — that belongs to Plan 03. This config is PWA-ready but not PWA-enabled yet.

    **Step 4: Create `pwa/src/app.html`:**
    ```html
    <!doctype html>
    <html lang="es">
    ```

    CRITICAL: `lang="es"` per UI-SPEC accessibility. `viewport-fit=cover` for notch/safe-area support on iOS.
