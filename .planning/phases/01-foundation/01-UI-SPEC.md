## Spacing Scale

Declared values (multiples of 4):

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--vsa-space-xs` | 4px | Icon gaps, inline padding, badge internal padding |
| sm | `--vsa-space-sm` | 8px | Compact element spacing, tab bar icon-to-label gap, onboarding dots gap |
| md | `--vsa-space-md` | 16px | Default element spacing, card padding, tab bar side padding |
| lg | `--vsa-space-lg` | 24px | Section padding, card internal spacing, onboarding slide padding |
| xl | `--vsa-space-xl` | 32px | Layout gaps, spacing between Home hero and stats card |
| 2xl | `--vsa-space-2xl` | 48px | Major section breaks, onboarding icon top margin |
| 3xl | `--vsa-space-3xl` | 64px | Page-level spacing (reserved for content phases) |

Exceptions:
- Tab bar height: 56px (44px touch target + 12px safe area padding for notch devices)
- Touch targets: minimum 44x44px for all interactive elements (WCAG 2.5.5)
- Header height: 48px
- Onboarding dots: 8px diameter, 8px gap between dots (`--vsa-space-sm`)

**Source:** CONTEXT.md Claude's Discretion (spacing system), RESEARCH.md Pattern 2 (proposed scale)

### Border Radius

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| sm | `--vsa-radius-sm` | 8px | Badges, small buttons, install button |
| md | `--vsa-radius-md` | 12px | Cards, tab bar items |
| lg | `--vsa-radius-lg` | 16px | Onboarding slides, bottom sheets, large cards |
| full | `--vsa-radius-full` | 9999px | Onboarding dots, pill buttons, CTA button |

Accent reserved for: primary CTA button ("Explorar el plan"), active tab indicator and icon, install button icon/text, onboarding active dot, "Empezar" button. Never for body text, card backgrounds, or decorative elements.

**Palette rationale:** Emerald green (`#10B981`) as accent provides a modern fintech aesthetic aligned with Revolut/Nubank/Cash App references. It avoids Venezuelan flag colors (yellow/blue/red) per D-05. The near-true-black backgrounds (`#09090B`) maximize AMOLED battery savings per D-08. Zinc-based neutrals (`#18181B`, `#A1A1AA`) provide a clean, app-native feel.

**Source:** CONTEXT.md D-05 (new palette, no Docusaurus colors, no flag colors), D-06 (app nativa moderna), D-08 (dark-first, AMOLED), Claude's Discretion (exact palette)

## Interaction Contracts

### Transitions

All transitions use CSS only. No JavaScript animation libraries (per REQUIREMENTS.md Out of Scope).

| Interaction | Property | Duration | Easing |
|-------------|----------|----------|--------|
| Tab switch | opacity, transform | 150ms | ease-out |
| Button hover/press | background-color, transform | 120ms | ease |
| Button press scale | transform: scale(0.97) | 100ms | ease-in-out |
| Card hover (desktop) | box-shadow, transform | 200ms | ease |
| Theme toggle icon | opacity | 200ms | ease |
| Onboarding slide | scroll-snap (native) | native | native |
| Pull-to-refresh spinner | opacity, transform | 200ms | ease-out |
| Page content fade-in | opacity | 200ms | ease-out |

### Touch Targets

All interactive elements: minimum 44x44px (WCAG 2.5.5). Tab bar items span full width divided by 4 (minimum ~90px on 360px screen) with 56px height.

### Scroll Behavior

- `overscroll-behavior-y: contain` on main content area (prevents browser pull-to-refresh conflict)
- `scroll-snap-type: x mandatory` on onboarding slides container
- `-webkit-overflow-scrolling: touch` for momentum scrolling on iOS

## CSS Token Implementation

All tokens live in `src/lib/styles/tokens.css` and are consumed via `var(--vsa-*)` throughout the app. The `:root` selector sets dark mode values (default). The `.light` class overrides for light mode. `mode-watcher` manages class application.

```
:root                    -> dark mode tokens (default)
:root.light              -> light mode overrides
```

No Tailwind. No CSS-in-JS. No runtime CSS generation. Zero-overhead native custom properties.

**Source:** RESEARCH.md Pattern 2, Alternatives Considered (vanilla CSS over Tailwind)

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Color contrast | All text meets WCAG AA (4.5:1 for body, 3:1 for large text). Accent on dark bg: `#10B981` on `#09090B` = 5.4:1. Accent on light bg: `#059669` on `#FFFFFF` = 4.7:1 |
| Touch targets | Minimum 44x44px for all interactive elements |
| Focus indicators | 2px solid accent outline with 2px offset on keyboard focus (`:focus-visible`) |
| Screen reader | All icons have `aria-label`. Active tab has `aria-current="page"`. Onboarding slides have `role="tabpanel"` |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` disables all transitions, sets `scroll-behavior: auto` |
| Language | `<html lang="es">` on all pages |
