<objective>
Wire inline source badge interactivity (CONT-05 gap closure).

Purpose: The remark-source-badges plugin emits static `<span class="vsa-source-badge" data-sources="...">` HTML that is visually present but interactively inert. SourceBadge.svelte and SourceSheet.svelte exist with correct implementations but are never imported. This plan adds event delegation on the article container to intercept badge clicks, parse the data-sources attribute, open a SourceSheet bottom sheet, and style the static badges with type-specific colors.

Output: Tapping any inline source badge opens a bottom sheet with full source details (organization, date, URL).
</objective>

  <action>
    Modify Article.svelte to:

    1. **Import** SourceSheet and browser check:
       ```js
       import { browser } from '$app/environment';
       import SourceSheet from './SourceSheet.svelte';
       ```

    2. **Add reactive state** for the sheet:
       ```js
       let sheetSources = $state([]);
       let sheetOpen = $state(false);
       ```

    3. **Add a ref** for the article element:
       ```js
       let articleEl = $state(null);
       ```

    4. **Add click handler function** that uses event delegation:
       ```js
       function handleArticleClick(e) {
         const badge = e.target.closest('.vsa-source-badge');
         if (!badge) return;

         const raw = badge.getAttribute('data-sources');
         if (!raw) return;

         try {
           const parsed = JSON.parse(raw);
           if (Array.isArray(parsed) && parsed.length > 0) {
             sheetSources = parsed;
             sheetOpen = true;
           }
         } catch (err) {
           // Malformed data-sources — silently ignore, ChapterSources fallback exists
         }
       }
       ```

    5. **Add keydown handler** for keyboard accessibility (Enter/Space on focused badge):
       ```js
       function handleArticleKeydown(e) {
         if (e.key !== 'Enter' && e.key !== ' ') return;
         const badge = e.target.closest('.vsa-source-badge');
         if (!badge) return;
         e.preventDefault();
         handleArticleClick({ target: badge });
       }
       ```

    6. **Add close handler**:
       ```js
       function closeSheet() {
         sheetOpen = false;
         sheetSources = [];
       }
       ```

    7. **Update the template** — add onclick/onkeydown to the article element, bind:this for the ref, and mount SourceSheet:
       ```svelte
       <!-- svelte-ignore a11y_no_static_element_interactions -->
       <article
         class="vsa-article"
         bind:this={articleEl}
         onclick={handleArticleClick}
         onkeydown={handleArticleKeydown}
       >
         {#each docs as doc, i}
           <section class="vsa-article__section" id="doc-{i}">
             <svelte:component this={doc.component} />
           </section>
         {/each}
       </article>

       <SourceSheet sources={sheetSources} open={sheetOpen} onclose={closeSheet} />
       ```

    The `articleEl` ref is kept for potential future use but not strictly needed since onclick is on the element directly. The key design: ONE SourceSheet instance mounted per Article, controlled by reactive state. Event delegation avoids the need to wrap each static badge span in a Svelte component.

    The svelte-ignore comment is needed because article is not an interactive element, but the click handler is delegation-based and the actual interactive elements (the badge spans) have role="button" and tabindex="0".
  </action>
