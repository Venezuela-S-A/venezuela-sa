# Phase 2 — UI Design Contract

> Visual and interaction contract for the content migration phase. All 85 markdown documents rendered as a navigable book with search, deep links, source attribution, and sequential chapter navigation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla CSS custom properties, inherited from Phase 1) |
| Preset | not applicable |
| Component library | none (custom Svelte 5 components with BEM `vsa-*` namespace) |
| Icon library | @lucide/svelte (already installed, tree-shakeable) |

**Source:** Phase 1 UI-SPEC (design system established), CONTEXT.md code_context (established patterns)

## Spacing Scale

Inherited from Phase 1. All tokens defined in `pwa/src/lib/styles/tokens.css`.

| Token | CSS Variable | Value | Usage in Phase 2 |
|-------|-------------|-------|-------------------|
| xs | `--vsa-space-xs` | 4px | Source badge internal padding, admonition icon gap, search result metadata gap |
| sm | `--vsa-space-sm` | 8px | Paragraph spacing within articles, chapter card badge gap, diagram caption gap |
| md | `--vsa-space-md` | 16px | Article body paragraph margin, chapter card padding, admonition padding, search bar padding |
| lg | `--vsa-space-lg` | 24px | Section spacing between docs within a chapter, chapter card internal spacing, source sheet padding |
| xl | `--vsa-space-xl` | 32px | Spacing between chapter landing hero and card grid, chapter nav bar padding |
| 2xl | `--vsa-space-2xl` | 48px | Spacing between chapter sections (within continuous scroll), collapsible sources section top margin |
| 3xl | `--vsa-space-3xl` | 64px | Top/bottom page padding on chapter reader, footer spacing before chapter nav |

Exceptions:
- Reading progress bar height: 3px (thin line at top of viewport, not a spacing token)
- Table fade shadow width: 32px gradient (matches `--vsa-space-xl` for visual consistency)
- Bottom sheet handle: 4px height, 36px width, centered
- Fullscreen diagram viewer toolbar: 48px height (matches `--vsa-space-2xl`)

**Source:** Phase 1 UI-SPEC (spacing scale), CONTEXT.md D-03 (progress bar), D-14 (table fade)

## Typography

Inherited from Phase 1. Four sizes, two weights.

| Role | CSS Variable | Size | Weight | Line Height | Phase 2 Usage |
|------|-------------|------|--------|-------------|---------------|
| Caption | `--vsa-font-size-xs` | 12px (0.75rem) | 400 | 1.4 | Source badge labels, reading time, progress percentage, chapter card badge text, diagram captions, search result URL, collapsible sources date |
| Body | `--vsa-font-size-base` | 16px (1rem) | 400 | 1.5 | Article body text, admonition content, chapter card descriptions, search input, search result excerpts, source sheet body, chapter nav labels |
| Subheading | `--vsa-font-size-lg` | 20px (1.25rem) | 600 | 1.3 | Article H2/H3 headings, chapter card titles, landing hero subtitle, search result titles, admonition titles, source sheet organization name |
| Display | `--vsa-font-size-2xl` | 28px (1.75rem) | 600 | 1.2 | Article H1 (chapter title at top of reader), landing hero headline, chapter card hook stat numbers |

### Article Typography (additional detail)

Markdown heading levels map to these tokens:

| Markdown | Element | Size | Weight | Margin Top | Margin Bottom |
|----------|---------|------|--------|------------|---------------|

| `## H2` | Major section heading | 20px subheading | 600 | 48px (`--vsa-space-2xl`) | 16px (`--vsa-space-md`) |
| `### H3` | Subsection heading | 20px subheading | 600 | 32px (`--vsa-space-xl`) | 8px (`--vsa-space-sm`) |
| `#### H4` | Minor heading (rare) | 16px body | 600 | 24px (`--vsa-space-lg`) | 8px (`--vsa-space-sm`) |
| `<p>` | Body paragraph | 16px body | 400 | 0 | 16px (`--vsa-space-md`) |
| `<li>` | List item | 16px body | 400 | 0 | 8px (`--vsa-space-sm`) |
| `<blockquote>` | Blockquote | 16px body | 400 | 16px | 16px |
| `<code>` inline | Inline code | 14px (0.875rem) | 400 | inherit | inherit |
| `<pre><code>` | Code block | 14px (0.875rem) | 400 | 16px | 16px |

**Source:** Phase 1 UI-SPEC (typography scale), CONTEXT.md D-01 (continuous scroll reading)

## Interaction Contracts

### Transitions

All transitions use CSS only. No JavaScript animation libraries.

| Interaction | Property | Duration | Easing |
|-------------|----------|----------|--------|
| Reading progress bar | width | 100ms | linear |
| Source sheet open | transform (translateY) | 200ms | ease-out |
| Source sheet close | transform (translateY) | 150ms | ease-in |
| Source sheet backdrop | opacity | 200ms | ease |
| Chapter nav hover/press | background-color | 120ms | ease |
| Diagram fullscreen open | opacity | 150ms | ease-out (native `<dialog>`) |
| Diagram fullscreen close | opacity | 150ms | ease-in |
| Search dropdown appear | opacity, transform (translateY 4px) | 150ms | ease-out |
| Search dropdown disappear | opacity | 100ms | ease-in |
| Chapter sources expand | max-height | 200ms | ease |
| Chapter sources collapse | max-height | 150ms | ease |
| Chapter card press | transform: scale(0.98) | 100ms | ease-in-out |
| Admonition icon | none (static) | - | - |
| Table fade shadow | opacity (scroll-triggered) | 100ms | ease |

### Touch Targets

All interactive elements: minimum 44x44px (WCAG 2.5.5).

| Element | Touch Area | Notes |
|---------|------------|-------|
| Chapter card | Full card surface | `--vsa-radius-md` active state background shift |
| Source badge | 44px height minimum (padding expands small badges) | Tap opens bottom sheet |
| Search input | 48px height | Comfortable text entry on mobile |
| Search result item | Full width x min 56px height | Padding ensures touch target |
| Chapter nav prev/next | 50% width x 56px height | Each half of the nav bar |
| Diagram expand button | 44x44px | Overlay on top-right of complex diagrams |
| Diagram fullscreen close | 44x44px | Top-right of fullscreen view |
| Source sheet dismiss (X) | 44x44px | Top-right of bottom sheet |
| Chapter sources toggle | Full width x 48px | Tap to expand/collapse |
| Continue Reading CTA | Full width x 48px | Pill button |

### Scroll Behavior

| Context | Behavior |
|---------|----------|
| Chapter reader | `overscroll-behavior-y: contain` (prevents browser pull-to-refresh conflict, inherited from Phase 1) |
| Table horizontal scroll | `-webkit-overflow-scrolling: touch` for momentum scrolling on iOS. `scroll-snap-type: none` (free scroll) |
| Fullscreen diagram | `touch-action: pinch-zoom` on SVG container. `overflow: auto` for panning |
| Landing page | Normal scroll. `scroll-behavior: smooth` for anchor navigation only |
| Search results | `max-height: 60vh` with `overflow-y: auto` |
| Continue reading resume | `window.scrollTo({ top: savedY, behavior: 'instant' })` -- no smooth scroll on resume (instant jump to position) |

### Deep Link Behavior

| Element | ID Pattern | Example |
|---------|-----------|---------|
| Heading (H1-H4) | github-slugger algorithm via rehype-slug | `#reservas-petroleras` |
| Paragraph | `p-{sequential-index}` within chapter | `#p-14` |
| URL format | `/plan/{chapter-slug}#{element-id}` | `/plan/fundamentos#reservas-petroleras` |
| Scroll on load | If URL has hash, scroll to element after render with 80px top offset (clearing fixed header + progress bar) |

## Accessibility

Inherited from Phase 1, plus:

| Requirement | Implementation |
|-------------|---------------|
| Article semantics | `<article>` wrapping each chapter reader. `<section>` wrapping each doc within. `<h1>` for chapter title, `<h2>`-`<h4>` for content headings |
| Navigation landmarks | `<nav aria-label="Capitulos">` for chapter nav. `<nav aria-label="Busqueda">` for search |
| Source badges | `role="button"` + `aria-label="Fuente: {org names}. Toca para ver detalles."` + `tabindex="0"` |
| Source sheet | `role="dialog"` + `aria-modal="true"` + `aria-label="Detalle de fuente: {org}"`. Focus trap when open |
| Diagram viewer fullscreen | `<dialog>` element (native). `aria-label="Diagrama: {caption}"`. `Escape` key closes |
| Admonitions | `role="note"` + `aria-label="{type}: {title}"` |
| Search | `role="search"` on container. `role="listbox"` on results. `aria-live="polite"` for result count announcement |
| Reading progress | `role="progressbar"` + `aria-valuenow={percent}` + `aria-valuemin="0"` + `aria-valuemax="100"` + `aria-label="Progreso de lectura"` |
| Deep link scroll | After scroll to hash target, focus the target element for screen reader announcement |
| Table wrapper | `tabindex="0"` + `role="region"` + `aria-label="Tabla con scroll horizontal"` on scrollable container |
| Reduced motion | All transitions respect `prefers-reduced-motion: reduce` (inherited from tokens.css) |
| Language | `<html lang="es">` (inherited) |
